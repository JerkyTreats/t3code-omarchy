# Docs Style Policy

Date: 2026-03-07
Status: active

## Intent

Keep docs concise, scannable, and operationally useful.

## Rules

- Do not use literal `(` or `)` in Markdown prose.
- Parentheses are allowed only for Markdown link syntax and inside inline code or fenced code blocks.
- Prefer short sections with direct headings and explicit decisions.
- When a diagram adds clarity, prefer Mermaid and keep it next to the related explanation.
- For architecture and workflow docs, name concrete owners, boundaries, and verification gates.
- For plans under `.plans/`, keep status and done criteria easy to scan.
